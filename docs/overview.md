# 项目概述

## 📋 项目简介

Twinkle Tools 是一个现代化的在线工具集网站，为开发者和设计师提供便捷、实用的日常工具服务。项目采用 Next.js 14 + TypeScript + Tailwind CSS + shadcn/ui 技术栈构建。

### 核心特性
- 🎨 **现代化设计** - 采用 Material Design 设计语言，界面美观直观
- 📱 **响应式布局** - 完美适配桌面端和移动端设备
- ⚡ **高性能** - 基于 Next.js 14 的 App Router 和服务端渲染
- 🔒 **安全可靠** - 本地数据处理，保护用户隐私
- 🌙 **深色模式** - 支持浅色/深色主题切换
- 🛠 **丰富工具** - 涵盖开发、设计、文本处理等多个领域

## 🎯 项目目标

### 主要目标
1. **用户体验优先** - 提供流畅、直观的使用体验
2. **功能实用** - 聚焦实际需求，提供真正有用的工具
3. **性能卓越** - 快速加载，流畅操作
4. **易于扩展** - 模块化设计，便于添加新工具

### 长期规划
- 逐步扩展工具类别，覆盖更多使用场景
- 增加用户数据同步功能
- 开发移动端应用
- 建立插件系统，支持第三方工具集成

## 🏗 架构设计

### 整体架构
```
┌─────────────────────────────────────────┐
│                 用户界面                 │
│    (Next.js + React + Tailwind CSS)     │
├─────────────────────────────────────────┤
│              组件层 (shadcn/ui)           │
├─────────────────────────────────────────┤
│               业务逻辑层                 │
│         (工具模块 + 页面路由)            │
├─────────────────────────────────────────┤
│              数据处理层                 │
│        (本地存储 + 工具算法)             │
└─────────────────────────────────────────┘
```

### 技术架构
- **前端框架**: Next.js 14 (App Router)
- **开发语言**: TypeScript
- **样式方案**: Tailwind CSS 4.0
- **组件库**: shadcn/ui (基于 Radix UI)
- **图标库**: Lucide React
- **运行时**: Bun

## 🎨 设计系统

### 颜色主题
```css
/* 主色调 */
--primary: hsl(222.2, 84%, 4.9%);      /* 深蓝色 */
--primary-foreground: hsl(210, 40%, 98%);

/* 辅助色 */
--secondary: hsl(210, 40%, 96%);       /* 浅灰色 */
--accent: hsl(210, 40%, 96%);          /* 强调色 */

/* 功能色 */
--destructive: hsl(0, 84.2%, 60.3%);   /* 危险色 */
--success: hsl(142, 76%, 36%);         /* 成功色 */
--warning: hsl(38, 92%, 50%);          /* 警告色 */
```

### 间距系统
采用 Tailwind CSS 的标准间距系统：
- `4px` (1) - 最小间距
- `8px` (2) - 小间距
- `16px` (4) - 默认间距
- `24px` (6) - 大间距
- `32px` (8) - 超大间距

### 字体系统
- **主字体**: Geist Sans (无衬线字体)
- **代码字体**: Geist Mono (等宽字体)
- **字体大小**: 12px - 48px 响应式缩放

## 🔄 开发流程

### 开发环境
- **Node.js**: 18+
- **Bun**: 作为包管理器和运行时
- **IDE**: 推荐 VS Code + 相关插件

### Git 工作流
```bash
# 功能开发
git checkout -b feature/new-tool
git commit -m "feat: add new tool feature"
git push origin feature/new-tool

# 发布流程
git checkout main
git merge feature/new-tool
npm run build
npm run start
```

### 代码质量
- **ESLint**: 代码规范检查
- **TypeScript**: 类型安全
- **Prettier**: 代码格式化
- **Husky**: Git hooks 管理
